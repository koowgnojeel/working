This message appears when
DarkIce lacks the necessary system permissions
to request high-priority CPU scheduling from the Linux kernel. 

Common Solutions

* Ignore the Warning
  In many cases, especially on modern hardware like a Raspberry Pi,
  this warning can be safely ignored
  as it rarely impacts stream quality.

* Run with Sudo
  Execute the command using sudo darkice
  to grant the program the super-user privileges
  required for POSIX real-time scheduling.

* Disable Real-time in Config
  Open your DarkIce configuration file (usually /etc/darkice.cfg)
  and set realtime = no under the [general] section
  to prevent the program from attempting high-priority scheduling.

* Adjust System Limits
  For a more permanent fix without using sudo,
  you can modify your system's security limits.
  Add the following lines to the Linux limits configuration at
  /etc/security/limits.conf

  * USERNAME soft rtprio 99
  * USERNAME hard rtprio 99 

When to Worry

If you notice actual audio skipping or stuttering in your live stream,
it indicates your CPU is struggling to keep up.
If the audio sounds clear, the warning is merely informational.

---

/etc/darkice.cfg
    [general]
    realtime = no
